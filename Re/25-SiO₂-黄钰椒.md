*** IDA

1. IDA Pro
是静态反汇编工具：它的核心作用就是把程序的二进制机器码（十六进制），还原成人类可读的x86/x64/ARM
等汇编语言。
​
- 比如截图里的  mov eax, \[ebp+8\] 、 add eax,
[ebp+0Ch\] ，就是IDA把机器码反汇编出来的结果。

![](media/image1.jpeg){width="5.75in" height="11.104166666666666in"}

2. F5 是「伪代码生成」功能：IDA 会把汇编代码进一步分析、优化，还原成接近
C 语言的高级代码，这就是我们说的「伪代码」。
​

**它不是程序原本的源代码，而是IDA根据汇编逻辑逆向推导出来的等价逻辑，所以叫「伪代码」**。

**维度** 真实源代码 IDA F5 伪代码\
**来源** 程序员手写的 文件 IDA 从汇编代码逆向推导出来的逻辑\
**信息完整性** 包含变量名、函数名、注释、**结构体定义**
丢失了所有符号信息，变量名是IDA自动生成的\
**语法准确性** 完全符合C/C++语法，可直接**编译**
语法接近C，但部分逻辑是IDA的近似还原，不能直接编译运行\
**优化影响** 编译器优化前的原始逻辑
编译器优化后，部分逻辑会被打乱，IDA只能尽力还原真实c代码

  -----------------------------------------------------------------------
  Bash
  int add(int a, int b) {\
  return a + b;\
  }

  -----------------------------------------------------------------------

编译后变成机器码 ida反汇编

  -----------------------------------------------------------------------
  Bash
  push ebp
  mov ebp, esp
  mov eax, [ebp+8\] ; 取第一个参数a
  add eax, [ebp+0Ch\] ; 加第二个参数b
  pop ebp
  retn

  -----------------------------------------------------------------------

生成的伪代码

  -----------------------------------------------------------------------
  Bash
  int \_\_cdecl add(int a, int b)\
  {
  return a + b;
  }

  -----------------------------------------------------------------------


逻辑完全一致，但变量名、函数名是IDA根据栈结构分析出来的，不是原代码里的名字。
​

如果编译器开了优化，甚至可能把函数内联，IDA就再也还原不出这个独立的 add 函数了。

**工具** 适用场景 核心作用\
**jadx** 安卓APK逆向
把Java字节码（）反编译成Java源代码，直接出源码级结果，不用F5\
**IDA Pro** 二进制逆向（exe/elf/so等）
把机器码反汇编成汇编，再F5成C伪代码，是Native层逆向的核心工具

**** 一、安卓APK到底是什么？

APK 就是安卓APP的安装包文件，**后缀名
 .apk ，本质是一个压缩包（你直接用解压软件就能打开）**。
它里面包含了APP运行需要的所有东西：代码、资源、图片、配置文件等。

APK里的代码分两种核心类型（这就是你问的Java/C的区别）：

**代码类型** 对应文件 运行环境 工具选择
**Java/Kotlin 代码** （核心） 安卓虚拟机（ART/Dalvik）
jadx、JEB、Apktool（直接反编译出Java代码）
C/C++ 代码（Native层） （在文件夹下） 直接运行在CPU上 IDA
Pro、Ghidra（反汇编成汇编，再F5转C伪代码）

**** 二、怎么判断APK里是Java还是C写的？

不用反编译，打开APK就能一眼看出来，完全不用猜：

方法1：解压APK，看文件结构

1. 把  .apk  后缀改成  .zip ，用解压软件打开
​
2. 看两个关键位置：
​
- 有 ** classes.dex  文件 → 一定有Java/Kotlin代码（99%的APK都有）**\
​
- 有 ** lib/  文件夹，里面有  .so  文件 → 一定有C/C++写的Native代码**\

绝大多数**安卓APP是混合开发：Java写界面和逻辑，C写加密、算法、核心逻辑（CTF题尤其爱这么出）**。\

方法2：用jadx直接打开APK（最省事）

你已经装了jadx，直接拖进去：

- 左侧能看到  smali / java  包结构 → 有Java代码
​
- **左侧  Resources  →  lib  文件夹下有  .so  文件 → 有C代码**

方法3：CTF题里的规律（直接用）

- 题目说「安卓逆向」「APK」，没提so → 90%是Java题，用jadx
​
- 题目提了「so文件」「Native层」「ARM汇编」→ 100%是C题，用IDA

- **题目给了APK+so → 混合题，先看Java逻辑，再用IDA逆向so**

-
C层（so）：相当于APP的「后台」，负责核心算法、加密、解密，CTF题的flag几乎都藏在这里\

* 汇编语言

汇编语言就是用助记符（比如MOV、ADD）来表示机器指令的低级编程语言。它比直接写二进制机器码要容易理解，但比高级语言更接近硬件。

主要特点：

直接控制CPU和内存

执行效率高

依赖具体的CPU架构

常见用途：

系统底层开发（内核、驱动）

逆向分析和安全研究

性能关键代码的优化

** 寄存器和内存的通信

汇编的作用就是操作CPU进行计算，CPU在计算的时候需要存储计算的值，**这些值的存储位置就是CPU的寄存器（可以理解为cpu计算的草稿纸）**，复杂的计算还需要内存辅助数据的存储，这里看下内存与CPU寄存器的配合方式。

*** 关于寄存器

CPU的通用寄存器有eax、ebx、ecx、edx等，不同架构提供的通用寄存器数量和名称有区别，这些通用寄存器就是可以给用户侧（汇编代码）自由使用的寄存器，其他的非通用寄存器如eip、esp、eflags等是为了CPU实现一些功能用的，不会开放给用户侧使用。

其中寄存器eax通常会当作返回值使用，比如我们的可执行文件执行完成返回的值（echo
\$?）其实就是执行完成后寄存器eax当时的值。

***** ZF

一、核心定义

ZF 是 x86/x64 汇编中的「零标志位」，**用来记录运算结果是否为 0**，是 CPU
状态寄存器（FLAGS/RFLAGS）里的一个 bit 位。

运算结果 = 0 → ZF = 1（置位）

运算结果 ≠ 0 → ZF = 0（复位）

本质：用一个 bit 标记「上一次运算有没有得到
0」，是所有条件跳转指令的核心判断依据。

二、最常用的触发指令

** test 寄存器, 寄存器 （最常见，判断是否为0）**

作用：对两个数做按**位与运算**，结果只影响标志位，不修改原寄存器

最常用写法： test eax, eax

若  eax = 0  → 按位与结果 = 0 → ZF = 1

若  eax ≠ 0  → 按位与结果 ≠ 0 → ZF = 0

逆向场景：判断函数返回值、指针是否为空、字符串是否为空

 cmp 操作数1, 操作数2 （最常用，判断大小/相等）

作用：做  操作数1 - 操作数2  运算，结果只影响标志位，不修改原数

对 ZF 的影响：

若  操作数1 == 操作数2  → 差为 0 → ZF = 1

若  操作数1 ≠ 操作数2  → 差不为 0 → ZF = 0

逆向场景：字符串比较（ strcmp  后）、数值校验、分支判断

其他运算指令（ add/sub/and/or/xor  等）

所有算术/位运算都会根据结果是否为0，自动修改 ZF：

运算结果为 0 → ZF=1

运算结果非 0 → ZF=0

*** 栈操作指令

栈操作指令用于管理函数调用和局部变量，是函数调用约定的重要组成部分

**** 栈操作

  -----------------------------------------------------------------------
  PHP\
  ; PUSH指令示例\
  push eax ; 将EAX压入栈\
  push 123 ; 将立即数123压入栈\
  \
  ; POP指令示例\
  pop eax ; 将栈顶元素弹出到EAX\
  pop ebx ; 将栈顶元素弹出到EBX

  -----------------------------------------------------------------------

**** 栈指针**

  -----------------------------------------------------------------------
  PHP\
  ; 栈指针调整\
  sub esp, 16 ; 分配16字节的栈空间\
  add esp, 16 ; 释放16字节的栈空间
