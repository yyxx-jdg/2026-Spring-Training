### 1）DOM树

结合刚才提到的 F12 开发者工具，**DOM 树**就是浏览器在内存中构建的那张**动态活地图**。

通俗理解它的三个关键词：

#### 1. 它是 HTML 的“结构化翻译版”

你把 **Ctrl+U** 看到的源代码想象成一份**用文字描述的装修图纸**（比如“左边有墙，右边有窗”）。
而 **DOM 树**是浏览器照着图纸，在脑海里真正**搭起来的那个立体房子的模型**。

- **HTML 源码**：`<div><p>Hello</p></div>` （纯文本）
- **DOM 树**：一个叫 `div` 的盒子，里面装着一个叫 `p` 的盒子，`p` 盒子里面有字 “Hello”。（对象结构）

#### 2. 它是“树状家族关系”

浏览器为了方便计算位置和样式，把网页里的每个标签、每段文字都看作一个**节点**，并按包含关系排成**家谱**：

- `<html>` 是**老祖宗**（根节点）
- `<head>` 和 `<body>` 是**大儿子和二儿子**
- `<div>` 是 `<body>` 的**孙子**

F12 里那个可以折叠、展开的缩进结构，就是这棵家族树的直观呈现。

#### 3. 它是“活”的（最关键的区别）

**Ctrl+U 的源代码是死的**，写啥就是啥。
**DOM 树是活的**，JS 代码可以像变魔术一样修剪它：

- 代码写的是空的 `<div id="app"></div>`（源代码里只有壳）。
- 但 DOM 树里，这个 `div` 下面已经长满了 Vue/React 生成的各种按钮和列表。

#### 总结一句话

**DOM 树 = 浏览器把原始代码 + JS 动态修改 + CSS 样式规则 全部揉在一起后，在内存里画出的、用来显示当前画面的“终极蓝图”。**

### 2）JS语言

虽然名字里都有 **Java**，但 **JavaScript** 和 **Java** 是两种**完全不同的编程语言**。它们的关系就像 **“雷锋”** 和 **“雷峰塔”**——除了名字像，本质毫无血缘关系。

结合刚才聊的 F12 和网页场景，它们的核心区别如下：

#### 1. 运行地盘不同
- **JavaScript (JS)**：活在**浏览器里**（现在也能活在服务器 Node.js 里）。它是 **F12 控制台** 里的王者，专门负责操作 **DOM 树**、让网页动起来。
- **Java**：主要活在**服务器后端、安卓手机、大型软件**里。**浏览器默认是跑不了 Java 的**（除非装插件，现代浏览器已不支持）。

#### 2. 语言类型不同（编译 vs 解释）
- **Java**：像**造一辆汽车**。需要先把 `.java` 源文件**编译**成 `.class` 字节码文件，然后交给 JVM（Java虚拟机）去跑。如果不编译，车轱辘都装不上。
- **JavaScript**：像**骑一辆自行车**。浏览器拿到 `.js` **源码直接解释执行**，不需要提前编译。你在 F12 里敲一行代码，回车立马出结果。

#### 3. 语法与规矩的严格程度
- **Java**：**强类型**，极其**严谨**。定义变量必须说清楚是放整数还是放文字，差一个分号都编译不过。
    ```java
    int number = 10; // 必须说清楚是 int (整数)
    number = "hello"; // ❌ 报错！Java 不让把字符串塞给整数变量
    ```
- **JavaScript**：**弱类型**，极其**灵活（随便）**。
    ```javascript
    let number = 10; // 现在是数字
    number = "hello"; // ✅ 完全没问题，秒变字符串
    ```

#### 4. 历史命名的乌龙
当初 JavaScript 诞生时，**Java 正如日中天**。为了蹭 Java 的热度好卖钱，Netscape 公司把原本叫 **LiveScript** 的语言临时改名成了 **JavaScript**。这纯粹是**市场营销手段**，导致几十年过去了大家还在解释它俩的区别。

#### 总结对照表

| 特征                    | **JavaScript (JS)**            | **Java**                           |
| :---------------------- | :----------------------------- | :--------------------------------- |
| **最常用场景**          | 网页前端特效、F12 改样式       | 企业级后端、安卓 App 开发          |
| **运行方式**            | 浏览器直接读代码执行（解释型） | 需要先编译成字节码再运行（编译型） |
| **语法风格**            | 自由散漫，怎么写都行           | 严谨刻板，一板一眼                 |
| **在刚才 F12 里的角色** | **它是画笔和锤子**（改 DOM）   | **它进不来这个门**                 |

### 3）JSS

结合刚才聊的 **F12**、**HTML** 和 **JS**，**CSS** 就是决定网页**长什么样**的“化妆师”和“装修队”。

如果用盖房子的比喻来串联前面提到的技术：

- **HTML** → 房子的**钢筋水泥框架**（哪里是卧室，哪里是门）。对应 `Ctrl+U` 看到的源代码。
- **DOM 树** → 房子盖好后**实际立在那里的物理结构**（对应 F12 里可编辑的元素）。
- **JavaScript** → 房子的**电路和开关**（按一下灯亮，再按一下门开了）。
- **CSS** → 房子的**装修、墙纸颜色、家具摆放位置**（卧室是粉色还是灰色，文字多大，动画效果）。

#### 在 F12 里最能直观感受到 CSS
你在 F12 开发者工具里点击任何一个小三角，都能看到一个 **Styles (样式)** 面板，里面全是一条条的 CSS 代码。它的作用就是告诉浏览器：

- 这个按钮**背景色**是蓝的。
- 这个字**大小**是 16 像素。
- 这个 div **离左边** 20 像素。
- 鼠标放上去**慢慢变透明**（过渡动画）。

#### 没有 CSS 的世界
你可以试着在浏览器按 `F12`，在右上角找到 **Network (网络)** 面板，勾选 **Disable CSS**（禁用样式），网页就会瞬间变成**纯文本流**——只有白底黑字蓝链接，没有任何布局、颜色和圆角。那就是**只有 HTML 骨架、没有 CSS 化妆**的样子。

#### 总结定义
**CSS** (Cascading Style Sheets，层叠样式表) 是专门用来**描述 HTML 元素如何显示**的语言。它不是编程语言（没有逻辑判断），而是一种**声明式配置语言**：

```css
/* 中文理解：我要把所有的 <p> 标签变成红色 */
p {
  color: red;
}
```

## 2.Hackbar

![image-20260418170035941](C:\Users\li130\AppData\Roaming\Typora\typora-user-images\image-20260418170035941.png)

#### 🛠️ 核心工作流

HackBar 的基本用法很简单，就是“加载-拆分-修改-执行”这个标准流程：

1. **加载 (LOAD)**：将当前页面的 URL 抓取到 HackBar 的编辑区，方便你修改参数。
2. **拆分 (SPLIT)**：把 URL 里的参数（比如 `?id=1&name=test`）按行拆开，让每个参数都一目了然。
3. **执行 (EXECUTE)**：在修改好参数后，用它来发送请求，并查看服务器的返回结果。

#### 💡 主要功能模块

在核心流程之外，HackBar 还提供了几个专项功能模块，用于处理各种特定的测试场景。

- **请求构建 (POST)**：勾选 `Enable Post Data`，你就能模拟提交表单数据，而不是只能在 URL 上做文章。
- **编码/加密 (ENCODING/ENCRYPTION)**：这是一组工具集，支持 **URL编码/解码**、**Base64编码/解码**、**MD5**、**SHA-1**、**SHA-256** 等常见的加解密算法，用于绕过过滤或数据处理。
- **注入辅助 (SQL/XSS/LFI等)**：
  - **SQL 注入**：可以生成标准的 `UNION SELECT` 语句，或把关键词转成十六进制（如 `admin` -> `0x61646d696e`）来绕过WAF防护。
  - **XSS 漏洞测试**：提供常用的攻击向量（payload），比如 `<script>alert(1)</script>`，方便快速测试。
  - **其他测试**：针对 LFI (本地文件包含)、SSRF (服务器端请求伪造) 等漏洞，HackBar 也提供了特定的测试功能。

### 2）核心功能标签页（菜单栏下方）

这是你日常使用最多的部分，从左到右逐个讲：

附加教程：[3.BP工具的基本使用操作_哔哩哔哩_bilibili](https://www.bilibili.com/video/BV1MMqGY2EP1?spm_id_from=333.788.videopod.episodes&vd_source=ccbcddb726666251ee446d44ceaed398&p=4)

**核心功能是：代理，重放器，intruder**

#### 1. 仪表盘（Dashboard）

- **核心作用**：项目的总控面板，相当于 “任务中心”。

- 主要功能

  ：

  - 查看当前正在运行的任务（比如扫描、爬虫）进度；
  - 实时监控请求日志、发现的漏洞告警；
  - 配置主动 / 被动扫描、爬虫任务的参数；
  - 查看整个项目的统计数据（请求数、漏洞数等）。

  

- **用法**：刚打开 Burp 时，默认就在这个页面，适合快速了解项目状态。

#### 2. 目标（Target）

- **核心作用**：定义测试范围、梳理目标网站的结构，是所有测试的基础。

- 关键子功能

  ：

  - **站点地图（Site map）**：自动记录所有访问过的 URL、参数、响应，形成网站结构树，帮你发现隐藏的接口和页面；
  - **Scope（范围）**：设置你要测试的目标域名 / IP，避免误扫其他网站，也可以过滤掉无关流量；
  - **内容发现（Content discovery）**：主动扫描目标网站的隐藏目录、文件（比如后台地址、备份文件）。

  

- **用法**：测试前先把目标域名加入 Scope，再用浏览器访问网站，站点地图会自动收录所有请求。

#### **3. 代理（Proxy）**

就是你当前打开的这个标签页！Burp 的**核心功能**，相当于浏览器和服务器之间的 “中间人”。

- **核心作用**：拦截、查看、修改浏览器与服务器之间的 HTTP/HTTPS 流量。

- 关键子标签（你界面上的）

  ：

  1. 拦截（Intercept）

     - 界面上的 `Intercept is off` 表示当前**不拦截**请求；点击后会变成 `Intercept is on`，此时浏览器的请求会被暂停在 Burp 里，你可以修改请求头、参数，再点 “放行” 发送给服务器。

     - 按钮功能：

       - `放行`：把修改后的请求发送给服务器；
       - `丢弃`：直接扔掉这个请求，服务器收不到；
       - `打开内嵌浏览器`：Burp 自带的浏览器，已经配置好代理，不用手动设置浏览器代理，直接抓包。

       

     

  2. **HTTP 历史记录**：所有通过 Burp 代理的请求都会在这里保存，按时间排序，可查看请求 / 响应详情，右键可发送到其他模块（比如 Repeater、Intruder）。

  3. **WebSocket 历史记录**：专门记录 WebSocket 协议的流量，适合测试实时通信类应用。

  4. **Match and replace**：自动修改请求 / 响应的内容，比如统一修改 User-Agent、添加请求头，不用每次手动改。

  5. **代理设置**：配置 Burp 的监听端口（默认 8080）、证书、代理规则，解决 HTTPS 抓包问题（需要安装 Burp 的 CA 证书）。

  

- 用法

  ：

  1. 配置浏览器代理到 `127.0.0.1:8080`；
  2. 开启 `Intercept is on`；
  3. 浏览器访问目标网站，请求会被拦截，修改后点 “放行” 即可。

  

#### **4. 重放器（Repeater）**

- **核心作用**：手动修改单个请求，反复发送给服务器，验证漏洞或测试接口行为。

- 用法

  ：

  1. 在 Proxy 的 HTTP 历史里，右键请求 → `Send to Repeater`；
  2. 切换到 Repeater 标签页，修改请求的参数、头信息、请求体；
  3. 点击`发送`，查看服务器返回的响应，对比修改前后的差异。

  

- **适用场景**：SQL 注入测试、XSS 验证、修改 Cookie 测试权限绕过等。

#### **5. Intruder（入侵者 / 爆破工具）**

- **核心作用**：自动化批量发送请求，对目标进行暴力破解、模糊测试。

- 关键功能

  ：

  - 支持多种攻击模式（Sniper、Battering ram、Pitchfork、Cluster bomb）；
  - 可自定义爆破位置（比如用户名、密码参数）；
  - 导入字典文件进行批量测试，自动分析响应结果（比如状态码、长度差异）。

  

- 用法

  ：

  1. 把要爆破的请求发送到 Intruder；
  2. 设置攻击位置（比如把密码参数标记为 payload 位置）；
  3. 导入字典，选择攻击模式；
  4. 启动攻击，查看响应结果，找出成功的请求。

  

- **适用场景**：密码爆破、用户名枚举、目录 / 路径探测、SQL 盲注批量测试。

#### 6. Collaborator（协作器）

- **核心作用**：生成 Burp 专属的域名 / 地址，用于检测服务器的带外交互（比如 SQL 盲注、XXE、SSRF 漏洞）。
- **用法**：在请求里插入 Collaborator 生成的地址，服务器访问这个地址时，Burp 会收到通知，证明漏洞存在。

#### 7. Sequencer（序列器）

- **核心作用**：分析数据样本的随机性，比如会话 Cookie、验证码、令牌是否可预测。
- **用法**：导入大量样本数据，Sequencer 会分析其随机性，如果随机性差，说明可能被暴力破解或预测。

#### 8. 编码工具（Decoder）

- **核心作用**：对数据进行编码 / 解码，支持 URL、Base64、HTML、Unicode 等格式，也支持哈希计算（MD5、SHA1 等）。
- **用法**：把请求里的编码数据粘贴进来，一键解码查看原文；也可以把明文编码后发送。

#### 9. 对比工具（Comparer）

- **核心作用**：对比两个请求 / 响应的差异，比如修改参数前后的响应变化，或者不同用户的页面差异。
- **用法**：选中两个请求，右键发送到 Comparer，工具会高亮显示差异部分。

#### 10. 日志（Logger）

- **核心作用**：记录 Burp 所有模块的请求和事件，相当于全局日志，方便排查问题。

#### 11. Organizer

- **核心作用**：整理和分类请求，方便后续复用和管理，相当于请求的 “收藏夹”。

#### 12. 扩展（Extender）

- **核心作用**：安装第三方插件，扩展 Burp 的功能，比如 SQL 注入扫描、绕过 WAF、导入其他工具的插件等。
- **用法**：加载 BApp Store 里的插件，或者本地安装第三方插件。

#### 13. 学习（Learning）

- **核心作用**：官方提供的教程和实验，帮助新手学习 Burp 的使用方法和 Web 安全知识。

# Linux基础学习笔记（核心定义+文件操作命令）

## 一、Linux基础认知

### 1. 什么是Linux

Linux是一套开源的、免费的操作系统内核，由林纳斯·托瓦兹于1991年开发，后续经全球开发者共同完善，形成了众多基于Linux内核的发行版（如Ubuntu、CentOS、RedHat等）。

核心特点：开源（源代码可自由查看、修改）、多用户、多任务、稳定性强、安全性高，广泛应用于服务器、嵌入式设备、云计算等领域，日常学习常用Ubuntu（图形化友好）或CentOS（服务器常用）。

### 2. Linux与Windows的核心区别

- 界面：Linux分图形界面（GUI）和命令行界面（CLI），命令行是学习和运维的核心；Windows以图形界面为主。
- 文件系统：Linux无“C盘、D盘”，以根目录（/）为起点，所有文件和目录都挂载在根目录下；Windows有盘符分区。
- 权限：Linux对文件和用户权限控制严格（读、写、执行）；Windows权限管理相对宽松。
- 开源性：Linux开源免费，可自由定制；Windows闭源收费。

## 二、Linux基础操作（命令行核心）

### 1. 命令行界面（终端）打开方式

Ubuntu：快捷键 Ctrl+Alt+T；CentOS：图形界面点击终端图标，或远程连接（如Xshell、Putty）。

命令行提示符含义（以Ubuntu为例）：`username@hostname:~$`，其中username是当前用户名，hostname是主机名，~是当前用户的家目录，$表示普通用户（#表示root管理员）。

## 三、文件与目录操作命令（核心重点）

Linux中“一切皆文件”，目录也是一种特殊文件，所有操作均通过命令实现，以下是最常用的文件读取、增删改相关命令，附带示例和说明。

### （一）目录相关基础命令（铺垫）

| 命令 | 作用                             | 示例                  | 说明                                                         |
| :--- | :------------------------------- | :-------------------- | :----------------------------------------------------------- |
| pwd  | 查看当前所在目录（打印工作目录） | pwd                   | 输出格式：/home/username（当前用户家目录）                   |
| cd   | 切换目录                         | cd /home、cd ~、cd .. | ..表示上一级目录，~表示家目录，/表示根目录                   |
| ls   | 列出当前目录下的文件/目录        | ls、ls -l、ls -a      | -l：详细信息（权限、大小、修改时间）；-a：显示隐藏文件（以.开头） |

### （二）文件读取命令（查看文件内容）

核心需求：查看文件中的内容，根据文件大小和查看需求选择不同命令。

| 命令 | 作用                                 | 示例                            | 关键说明                                                     |
| :--- | :----------------------------------- | :------------------------------ | :----------------------------------------------------------- |
| cat  | 一次性显示整个文件内容（适合小文件） | cat test.txt                    | 直接输出文件所有内容，若文件过大，内容会快速滚动             |
| more | 分页显示文件内容（适合大文件）       | more test.txt                   | 按空格键翻页，按q键退出查看                                  |
| less | 分页显示，可上下滚动（比more更灵活） | less test.txt                   | 按↑↓键滚动，按q退出，支持搜索（输入/关键词，按n下一个）      |
| head | 查看文件前N行（默认前10行）          | head test.txt、head -5 test.txt | -5表示查看前5行，适合快速查看文件开头                        |
| tail | 查看文件后N行（默认后10行）          | tail test.txt、tail -3 test.txt | -3表示查看后3行，常用查看日志文件（实时查看用tail -f 文件名） |

### （三）文件创建命令（增）

核心需求：新建空白文件、新建目录。

| 命令  | 作用                                         | 示例                              | 说明                                                         |
| :---- | :------------------------------------------- | :-------------------------------- | :----------------------------------------------------------- |
| touch | 创建空白文件，若文件已存在，更新文件修改时间 | touch test.txt、touch a.txt b.txt | 可同时创建多个文件，空格分隔                                 |
| mkdir | 创建目录（文件夹）                           | mkdir test、mkdir -p test1/test2  | -p：递归创建多级目录（如test1不存在，会先创建test1，再创建test2） |

### （四）文件删除命令（删）

核心需求：删除文件、删除目录，注意：Linux删除命令不可逆，谨慎操作！

| 命令  | 作用                               | 示例                        | 关键注意                                                     |
| :---- | :--------------------------------- | :-------------------------- | :----------------------------------------------------------- |
| rm    | 删除文件                           | rm test.txt、rm -f test.txt | -f：强制删除，不提示确认（避免删除时弹出“是否删除”的提示）   |
| rmdir | 删除空目录（只能删除空的）         | rmdir test                  | 若目录中有文件/子目录，无法删除，需先删除内部内容            |
| rm -r | 递归删除目录及内部所有内容（常用） | rm -r test、rm -rf test     | -r：递归删除；-rf：强制递归删除（高危命令，避免误删根目录！） |

警告：rm -rf / 是致命命令，会删除根目录下所有内容，导致系统崩溃，绝对禁止执行！

### （五）文件修改命令（改）

核心需求：修改文件名、移动文件/目录、修改文件内容、复制文件/目录。

| 命令 | 作用                                   | 示例                                                         | 说明                                                         |
| :--- | :------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| mv   | 移动文件/目录，或重命名文件/目录       | 1. 重命名：mv test.txt new.txt；2. 移动：mv test.txt /home/username | 移动时，若目标目录存在，文件会移动到该目录；若目标文件存在，会覆盖（无提示） |
| cp   | 复制文件/目录                          | 1. 复制文件：cp test.txt /home；2. 复制目录：cp -r test /home | -r：复制目录时必须加，递归复制目录内所有内容；-f：强制覆盖目标文件 |
| vim  | 文本编辑器，修改文件内容（核心编辑器） | vim test.txt                                                 | 操作步骤：1. 输入vim 文件名，进入编辑界面；2. 按i键进入插入模式（可修改内容）；3. 修改完成后，按Esc键退出插入模式；4. 输入:wq保存并退出（:q!强制退出不保存） |
| echo | 向文件中写入内容（覆盖或追加）         | 1. 覆盖写入：echo "hello linux" > test.txt；2. 追加写入：echo "hello linux" >> test.txt | >：覆盖文件原有内容；>>：在文件末尾追加内容，不覆盖原有内容  |

## 四、常用补充命令（必记）

- clear：清空终端屏幕（快捷键Ctrl+L也可）
- man 命令：查看命令的帮助文档（如man ls，查看ls命令的用法）
- whoami：查看当前登录的用户名
- pwd：再次强调，查看当前所在目录（避免迷路）

## 五、学习注意事项

1. 命令区分大小写：Linux命令严格区分大小写（如ls和LS是两个不同命令），必须小写输入。
2. 路径问题：切换目录、操作文件时，需注意路径是否正确（绝对路径：从/根目录开始；相对路径：从当前目录开始）。
3. 权限问题：若操作文件/目录提示“Permission denied”（权限不足），在命令前加sudo（管理员权限），如sudo rm test.txt（输入用户密码即可，密码输入时不显示）。
4. 多实操：Linux命令靠练习记忆，每学一个命令，立即在终端实操，避免“一看就会，一输就错”。
